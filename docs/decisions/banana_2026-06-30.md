# Decisions: Banana or No Banana — v1 MVP
Spec: Planning/specs/banana_spec.md
Date: 2026-06-30

- Adopted the repo-root `SPEC.md` as the spec, converted into `Planning/specs/banana_spec.md` — the brief named SPEC.md as the authoritative scope source of truth, so the spec was translated into Plumbline format (observable, tagged Done-when) rather than re-derived from the one-paragraph brief. PROPOSAL.md kept as context only.
- Organized the MVP as **6 feature blocks** (setup/coin, Ollama proxy + thinking-strip, exchange/turns, lock-in/scoring/reveal, logging, frontend+host) — mirrors SPEC's components and the build order while staying at the size threshold, not over it.
- Thinking-strip specced as **both** `think:false` **and** defensive `<think>…</think>` removal — SPEC calls it "the one true landmine"; a single mechanism was judged insufficient, so both are required and the strip is an `[automated]` unit-level criterion.
- Automated tests target a **mocked/faked Ollama endpoint**; a live `qwen3:8b` is required only for `[human-required]` playthrough items — keeps the core logic (coin fairness, turn accounting, scoring, parsing, log schema, strip) provable without a model in the loop, while the real-model feel stays human-judged. Assumption, non-blocking.
- Coin fairness made an observable criterion (±5% over ≥1000 seeded rounds) rather than "flips a coin" — the whole experiment is deviation from a 50% baseline, so the baseline itself must be provably fair.
- Scoring/parsing kept **mechanical** (no LLM judge) with `422` on unparseable `FINAL ANSWER` — clean pass/fail and frees the host to be pure showbiz; rejected silent scoring of ambiguous input.
- Filled CLAUDE.md Stack/Commands: FastAPI + httpx + uvicorn + vanilla JS + Ollama; run via uvicorn on 127.0.0.1:8000; test via pytest; UI evidence tool = playwright (python) because the spec has a browser stage. No stack fork remained (SPEC settled it), so nothing was routed to Open Questions.

## Update 2026-06-30 — Match settings feature (spec change, Full Path)
- Added a **Match settings** feature (user request): a settings panel to configure, per round, the Box Holder model, turn limit, and temperature. Previously the settings UI was deferred (SPEC §12); this pulls the model/turns/temperature slice of it into v1. The other seats, leaderboard, etc. remain deferred.
- **Model list auto-sourced** from Ollama `/api/tags` via a new `GET /api/models` endpoint (returns installed names + configured default) — chosen over a free-text field so only valid, installed models can be picked (user decision).
- **Per-round overrides**, not persisted: `POST /api/round` accepts optional `{model, turn_limit, temperature}`; `config.json` stays the baseline default. Rejected persisting settings to disk for v1 (keeps state in-memory, matches the one-active-round model). Guesser seat stays human (not configurable in v1).
- Bad settings (`turn_limit` < 1, non-numeric/negative `temperature`) → `422`, no round created — mechanical validation, consistent with the guess parser's `422`.
