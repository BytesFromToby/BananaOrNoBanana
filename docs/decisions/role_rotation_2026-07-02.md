# Decisions: Red/Blue players & role rotation
Spec: Planning/specs/banana_spec.md  (feature "Red/Blue players & role rotation", added 2026-07-02)
Date: 2026-07-02

- **Players become persistent color identities (RedPlayer / BluePlayer); role is assigned per game, not welded to the seat.** The old model fixed Left = Box Holder and Right = Guesser. Under a color identity the same contestant can hold *or* guess, which is what a fair matchup needs. Rejected keeping fixed roles — it can only ever measure one direction of a Red-vs-Blue matchup.

- **AI-vs-AI role rotation: alternate Box Holder/Guesser every completed, logged game; in-memory per-process state.** Fixed assignment biases the deviation-from-50% metric (a model measured only as holder, never as guesser). Alternating cancels role-assignment bias so a matchup measures both directions. Rejected per-round randomization (uneven over short runs — an even batch would not guarantee equal holds/guesses, which one Done-when requires) and persisted rotation state (rounds are already ephemeral and one-at-a-time per §10; a restart reset is acceptable and simpler).

- **One human present: the human *chooses* their role (`HUMAN_ROLE` = guesser | holder, default guesser); no auto-rotation.** Rotating a human's role between games would be disorienting and pointless — there is no bias to cancel when a person is in the loop. Default `guesser` preserves the established v1 default seat (the detective seat is the fun one, SPEC §13). At most one player is human.

- **Two-human party mode is a permanent non-goal.** Every game has at least one AI player — human-vs-AI (either role) or AI-vs-AI. Recorded as a hard `Does NOT` in Scope (2026-07-02). Keeps the no-leak invariant meaningful (a machine can be trusted not to peek at box_contents through a side channel; two humans sharing a browser cannot) and keeps role logic to the two cases that matter.

- **Human Box Holder path is a new `POST /api/round/{id}/hold {text}` endpoint mirroring `/advance` with roles reversed; `prompts/box_holder.md` is unused on this path.** The human bluffs in their own words, so no Box Holder system prompt is filled; each `/hold` call feeds the human's line to the AI Guesser (`prompts/guesser.md`) and returns a continuation or a `FINAL ANSWER` lock-in. `box_contents` is revealed to the human holder (they are allowed to know) but the Guesser-facing no-leak invariant is unchanged: no Guesser-facing response or streamed Guesser content carries `box_contents`. Rejected overloading `/say` (that endpoint is the human-Guesser path; on a human-holder round `/say` is rejected). This builds the previously-deferred roadmap item 6.

- **Hard rename `LEFT_PLAYER_*` → `RED_PLAYER_*`, `RIGHT_PLAYER_*` → `BLUE_PLAYER_*`; the local `.env` is migrated in place.** Single local user, no long back-compat window, so a clean rename beats carrying dual-name support. Touches players.py, game.py, app.py, log.py and the seat editors; the change is mechanical and cheap to revise if a name is wrong — hence an Assumption, not a fork.

- **The per-round `model` override (Match settings) is retired, not reinterpreted.** Under rotation "the Box Holder's model" is ambiguous (either color may hold), so each color's model comes from its own config. Reinterpreting the override would need an ambiguous rule about which color it applies to; removing it is unambiguous. `turn_limit`/`temperature` standard-settings are unchanged. The superseded seat feature's "LeftPlayer's model remains overridable per round" no longer holds.

- **Log records `holder_color` and `guesser_color` alongside the existing per-role provider/model.** Leaderboard aggregation already groups by box_holder-role × guesser-role model, so it captures rotation with no aggregation change; the color fields are additive and do not alter the shared arena wire format (`server/stats.py` / `arena/src/lib/aggregate.js` on `tests/fixtures/arena_rounds.json` stay in sync — new fields, no removed/renamed ones).

- **Pending: restructure (size flag, non-blocking).** The monolithic `Planning/specs/banana_spec.md` now carries ~10 `## Feature:` blocks — past the 6-feature threshold. architect never splits; graduating to per-feature specs + a reference tier is the manual step CLAUDE.md describes. Recorded here so the forecast isn't lost; it does not block this build.
